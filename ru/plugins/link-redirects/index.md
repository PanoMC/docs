# Плагин Link Redirects

Плагин **Link Redirects** позволяет создавать короткие красивые URL на вашем собственном сайте — например `yoursite.com/discord` — которые перенаправляют посетителей на любой внешний адрес, такой как `https://discord.gg/pano`. Перенаправления могут быть мгновенными или показывать фирменную страницу обратного отсчёта «Redirecting...» с выбором дизайнов (включая полностью настраиваемую HTML-страницу). Каждое перенаправление при желании может появляться как ссылка в навигационном меню вашего сайта и быть ограничено авторизованными пользователями или конкретным узлом прав доступа. Это официальный плагин Pano, бесплатный в использовании и лицензированный под MIT.

## Возможности

- **Неограниченные перенаправления:** сопоставьте любой путь сайта (должен начинаться с `/` и быть уникальным) с целевым URL.
- **Мгновенное перенаправление:** если не настроены задержка или промежуточная страница, посетители перенаправляются на стороне сервера (HTTP 302) — они никогда не видят пустую страницу.
- **Промежуточная страница «Redirecting»:** при желании показывайте страницу обратного отсчёта с настраиваемой задержкой (в секундах) и одним из четырёх дизайнов:
  - **Default:** белая карточка со спиннером, целевым URL и обратным отсчётом.
  - **Minimal:** спиннер с одной строкой — «Redirecting to {hostname} in {n}s».
  - **Modern:** тёмная карточка с анимированной полосой прогресса.
  - **Custom:** ваше собственное содержимое страницы, создаваемое в rich-text-редакторе панели с режимами HTML-исходника и предпросмотра. Наложение обратного отсчёта остаётся поверх.
- **Показ в навигации:** добавьте перенаправление в навигационное меню вашей темы, используя его заголовок в качестве текста ссылки, с опцией **Open in new tab** для каждой ссылки.
- **Контроль доступа:** для каждого перенаправления — **Require login** (анонимные посетители отправляются на `/login?redirect=<path>`) и **Require permission** с произвольным узлом прав доступа (например, `pano.custom.perm`). Неавторизованные посетители получают ответ «доступ запрещён»/404.
- **Удобства панели:** таблица с пагинацией (10 на страницу), показывающая ID, заголовок, адрес, цель и задержку; действия в строке для просмотра пути и цели, **Copy Link**, **Edit** и **Delete** (с модальным окном подтверждения); плюс тосты об успехе.
- **Аудит:** создание, обновление и удаление перенаправления записывается в журнал активности панели Pano.
- **Локализация:** доступно на английском, турецком и русском.

## Управление перенаправлениями

Пункт **Link Redirect** (иконка звеньев цепи) появляется в боковой панели сразу после **Posts** и открывает страницу Link Redirects. Всё управляется из единого модального окна **Create / Edit** — отдельной страницы настроек нет, как и общесайтовых опций; каждая настройка задаётся для отдельного перенаправления. Поля задержки и дизайна активны только при включённом **Show intermediate page**, редактор пользовательского содержимого появляется, только когда дизайн установлен в **Custom**, поле узла прав доступа появляется только при включённом **Require permission**, а **Open in new tab** активно только при включённом **Show in navigation**.

::: tip Бесплатно и официально
Link Redirects разрабатывается и поддерживается командой Pano, не требует премиум-лицензии и не имеет внешних зависимостей — ничего, кроме работающей установки Pano, не нужно.
:::

## Необходимое право доступа

Чтобы управлять перенаправлениями в панели, пользователям нужно право **Edit Redirects** (`MANAGE_REDIRECTS`) — «Can create, delete and edit a redirect which will be used in theme.». Оно ограничивает как пункт в боковой панели / страницу панели, так и все эндпоинты управления. Обратите внимание, что это отдельно от узла прав доступа для отдельного перенаправления, которым посетитель может быть обязан обладать.

## Настройка

1. Включите плагин в **панели администратора Pano**.
2. Перейдите в **Panel → Link Redirect**.
3. Нажмите **Create**, введите заголовок и путь (начинающийся с `/`) плюс целевой URL и сохраните.
4. При желании добавьте страницу обратного отсчёта, покажите ссылку в навигации или ограничьте доступ.
