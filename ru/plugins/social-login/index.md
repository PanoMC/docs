# Плагин Social Login

Плагин **Social Login** позволяет посетителям входить на ваш сайт на базе Pano через популярные социальные платформы вместо пароля. Он добавляет вход по OAuth 2.0 для **Google, Discord, Facebook, X (Twitter), GitHub, Twitch и Instagram** — посетители могут войти или зарегистрироваться в один клик, существующие участники могут привязывать и отвязывать эти аккаунты к своему профилю, а администраторы настраивают каждого провайдера из панели. Всё работает напрямую с официальным OAuth API каждого провайдера, без стороннего сервиса-посредника. Разработано командой Pano, бесплатно в использовании и с открытым исходным кодом.

## Возможности

- **Семь провайдеров:** Google, Discord, Facebook, X (Twitter), GitHub, Twitch и Instagram — каждый включается независимо.
- **Вход и регистрация:** фирменные кнопки «Continue with {provider}» появляются как на странице входа, так и на странице регистрации всякий раз, когда включён хотя бы один провайдер.
- **Умные потоки входа:** вернувшиеся пользователи с привязанным аккаунтом входят сразу; новые посетители получают страницу «Create Your Account» для выбора имени пользователя и пароля; а если у их email уже есть аккаунт, им предлагается один раз войти, чтобы привязать его.
- **Поддержка step-up:** шаг завершения обрабатывает дополнительные проверки входа, такие как коды 2FA или капча из других плагинов (например, двухфакторный запрос auth-guard).
- **Привязка аккаунтов:** карточка «Social Accounts» в настройках пользователя позволяет участникам привязывать или отвязывать провайдеров, а в их профиле отображается карточка «Linked Social Accounts» только для чтения.
- **Управление администратором:** просматривайте и отвязывайте подключённые аккаунты любого игрока с его страницы деталей в панели.
- **Понятные ошибки:** чёткие локализованные сообщения об отменённых входах, истёкших ссылках, уже привязанных аккаунтах и многом другом. Доступно на английском, турецком и русском.

## Конфигурация

Все настройки находятся на собственной странице деталей плагина в **панели администратора Pano** (Plugins → Social Login) — отдельного меню нет.

- **Для каждого провайдера:** переключатель включения, **Client ID**, **Client Secret** и кнопка копирования точного **Redirect URI** для регистрации. У каждого провайдера также есть встроенное пошаговое руководство по настройке с необходимыми scope и прямыми ссылками на консоль разработчика этого провайдера и его официальную документацию.
- **Глобальные настройки:** **Redirect Base URL** (ваш публичный origin, например `https://example.com`; оставьте пустым, чтобы использовать URL сайта платформы) и переключатель **Debug Logging** для пошаговой диагностики OAuth.
- **Обработка секретов:** сохранённый Client Secret маскируется (`••••`). Для его раскрытия требуется повторный ввод вашего пароля администратора, а если оставить поле нетронутым, сохранённое значение сохраняется.

## Необходимые права доступа

- **Manage Social Login Settings** — настройка провайдеров и их учётных данных OAuth.
- **Manage Social Links** — просмотр и отвязка подключённых аккаунтов игроков.

Элементы управления панели ограничены этими правами доступа.

## Предварительные требования

Прежде чем включить провайдера, вы должны создать приложение OAuth в консоли разработчика этого провайдера и зарегистрировать точный redirect URI, показанный в панели:

`{your-site}/api/social-login/{provider}/callback`

Ваш сайт должен быть доступен по публичному URL. У нескольких провайдеров есть дополнительные требования, на которые указывает встроенное руководство:

- **Facebook:** приложение должно быть переключено в режим **Live**.
- **Instagram:** требует приложение Meta **Business** с продуктом Instagram API; до одобрения App Review войти могут только добавленные тестировщики.
- **X (Twitter):** требует настройки OAuth 2.0 **Web App** (конфиденциальный клиент).

::: tip Бесплатно и официально
Social Login разрабатывается и поддерживается командой Pano. Он не требует премиум-лицензии и аккаунта panomc.com и дожидается завершения мастера первичной настройки Pano перед активацией.
:::

## Настройка
1. Включите плагин в **панели администратора Pano**.
2. Откройте страницу деталей плагина **Social Login**.
3. Для каждого нужного провайдера создайте приложение OAuth в его консоли разработчика, зарегистрируйте показанный **Redirect URI**, затем вставьте **Client ID** и **Client Secret** и включите его.
4. Задайте **Redirect Base URL**, если ваш публичный origin отличается от URL сайта вашей платформы, затем сохраните.
