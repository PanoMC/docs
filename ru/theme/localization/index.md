# Локализация (i18n)

Локализация (часто сокращается до **i18n**) означает отображение текста вашей темы на разных языках. Хорошая новость: движок Pano уже поставляется с полными переводами, поэтому вы записываете только те части, которые действительно меняете.

## Как это работает

Движок поставляется с полными переводами для языков, которые он поддерживает сегодня, — **en-US** (английский), **tr** (турецкий) и **ru** (русский). Вы **не** копируете и не поддерживаете эти большие файлы.

Вместо этого ваша тема несёт только **различия** в папке `lang-overrides/`, по одному файлу на язык:

```
lang-overrides/
├─ en-US.json
├─ tr.json
└─ ru.json
```

Когда вы запускаете `bun run sync`, Pano сливает ваши переопределения поверх переводов движка. Слияние **только добавляющее**: вы можете добавить совершенно новый текст или заменить существующий, но никогда не можете потерять ключ движка. Всё, что вы не упомянули, сохраняет значение движка по умолчанию.

Так что ваша работа невелика: пишите только те ключи, которые хотите изменить или добавить.

## Пример 1 — Изменить существующий текст

Допустим, вы хотите, чтобы подвал (footer) на турецком говорил что-то другое. Вам не нужно трогать файлы движка — вы просто переопределяете этот один ключ.

Откройте `lang-overrides/tr.json` и добавьте ключ, который хотите изменить:

```json
{
  "footer": {
    "copyright": "Benim harika sunucum tarafından yapıldı"
  }
}
```

После `bun run sync` турецкие посетители видят там ваш текст, а все остальные ключи подвала по-прежнему приходят из движка нетронутыми. Чтобы изменить ту же надпись на английском, добавьте её также в `en-US.json`; чтобы изменить на русском — в `ru.json`.

## Пример 2 — Добавить совершенно новый ключ

Если вы добавили собственную разметку — скажем, слоган-герой (hero slogan) в переопределённом представлении — вы можете добавить для него новый ключ перевода.

**1. Используйте ключ в представлении.** В вашем файле `.svelte` функция `$_` ищет ключ внутри пространства имён (namespace) вашей темы:

```svelte
<h1>{$_("my-theme.hero-slogan")}</h1>
```

**2. Добавьте ключ для каждого языка** в `lang-overrides/`:

```json
// lang-overrides/en-US.json
{
  "my-theme": {
    "hero-slogan": "Your adventure starts here"
  }
}
```

```json
// lang-overrides/tr.json
{
  "my-theme": {
    "hero-slogan": "Maceran burada başlıyor"
  }
}
```

```json
// lang-overrides/ru.json
{
  "my-theme": {
    "hero-slogan": "Твоё приключение начинается здесь"
  }
}
```

Теперь слоган автоматически показывается на языке посетителя. Запустите `bun run sync` после добавления ключей, чтобы слияние их подхватило.

::: tip Пропущен перевод?
Если у ключа нет перевода, на экране вместо реального текста показывается сырое имя ключа (например, `my-theme.hero-slogan`) — простой способ заметить проблему. `bun run check` также предупреждает вас о ключах, которые используют ваши представления, но которые не переведены, так что вы поймаете их до выпуска. См. [Упаковка](/ru/theme/packaging/).
:::
